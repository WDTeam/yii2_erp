关于系统错误码，我们设想，按照如下思路来做，首先按照各个子系统分类，定义系统所有的错误码，

整个家庭保洁的错误码，按照功能的不同分类，共分为4类，

1）客户端,1xxx
2) API,2xxx
3) Boss, 3xxx
4) 阿姨端,4xxx.

每个错误码都必须是4为数字。例如客户端的所有错误码，第一位是1， 第二位是功能的分类， 第三，四位可以是更具体的数字。
这样的设计，便于开发工程师，测试工程师，运维工程师，客户服务人员， 用户，阿姨之间便于沟通。
